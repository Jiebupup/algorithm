随着互联网规模的扩大，互联网上收集的计算资源、存储资源、数据资源和应用程序资源的数量不断增加，互联网正从传统的通信平台转变为无处不在的智能计算平台。为了支持对互联网资源的有效管理和综合利用，云计算技术应运而生。
云计算是一种基于互联网的计算方法。通过这种方式，可以使用服务提供商提供的用于计算和资源的计算机基础结构，根据需要将共享的软件和硬件资源以及信息提供给各种计算机终端和其他设备。用户不再需要了解云中基础架构的细节，不必需要具有相应的专业知识，也无需直接控制。云计算描述了一种新型的基于互联网的，用于添加、使用和交付的IT服务模型。它通常是动态的，并且易于在互联网上扩展，通常涉及提供虚拟化资源。
随着移动互联网的飞速发展和移动应用的极大丰富，移动终端已经成为人们工作生活不可或缺的一部分。然而，由于重量、大小和散热等因素的限制，移动终端的计算、存储资源与传统的非移动设备相比一直存在很大的差距，特别是移动终端有限的电池容量，严重影响了用户的应用体验。为了突破移动终端计算、存储和电池等资源限制，为移动用户提供更加丰富的应用，可以将云计算引入移动环境，移动云计算作为一种支撑移动云服务的关键技术应运而生。
移动云计算是指移动终端通过无线网络，以按需、易扩展的方式从云端获得所需的基础设施、平台、软件等资源或信息服务的使用与交付模式。《这个定义来自“移动云计算研究进展与趋势”，作者崔勇，你在参考文献上加上这个中文文献，这里引用上》移动应用程序将计算和数据存储从移动终端迁移到云端，它将云服务提供给使用各种类型移动终端的用户。

在车载云环境下，为了实现资源共享以及协同工作，如何进行合适的任务调度显得格外重要。首先，移动用户向车载云提交自己的请求。然后，车载云对数据进行处理，即任务执行。最后，车载云系统把结果返回给用户。但是，当多个用户在同一时间提出多个请求时《这不是你的研究问题，你的问题是单用户的一个application请求》，由于车载云的带宽资源和计算资源有限，可能会导致用户等待时间过长甚至用户的请求无法完成。《标黄的内容，去掉》本文的研究目的是对多任务进行调度，尽可能确保任务完成时间不超时，并且尽可能减小服务成本。
另外，由于车载云中车辆节点的移动性，车辆件数据传输率会随时间不断变化。并且还存在车辆节点离开云服务区，任务无法分配的场景。因此，车载云计算的任务调度算法在设计上与传统云计算的任务调度算法相比需要考虑更多的因素，传统算法无法满足车载云计算的需求。在考虑车载云计算中动态传输特性的条件下，设计出一个能够适用于车载云环境的任务调度算法对促进移动云计算技术的发展具有重要意义。
国内外研究的技术现状分析
在当前云计算环境下，用户对任务调度的QoS要求各异，任务数量和资源数量庞大，其任务调度问题比较复杂。传统云计算任务调度的QoS目标约束条件包括任务的截止时间、调度预算、资源利用率等。由于车载节点的移动特性，使得车载云计算和传统云计算相比还是存在很大的不同，车载云计算的任务调度问题是一个新颖且更复杂的研究方向。目前在这个方向上的研究还比较少。
Fei Sun等人基于遗传算法《注意引用的格式》提出采用整数编码，和使用“亲戚”的概念来避免不可行解，对车载云下的任务调度进行了研究，成功解决了复杂的车载云任务调度问题[1]。但是这种基于遗传算法的方法速度较慢，并且文中使用“时隙”来生成调度方案会造成服务时间的浪费。
Liang Bao等人在微服务的场景下，提出一种启发式的任务调度方案[4]，该方法是一种迭代处理过程，每次迭代中都重新计算关键路径，并且基于关键路径分配任务，通过迭代分配和延迟对比来减少任务完成时间。分配的方式仍然是传统的贪心算法。 
基于社会学中的信任关系模型以及经典的贝叶斯认知模型，有学者建立了一种新颖的基于认知信任关系模型的贝叶斯方法，并在此基础上提出了一种基于信任关系模型的动态优先级云计算任务调度算法（Cloud-DLS）；理论分析以及实验测试结果均表明，该云计算任务调度算法可以有效减少用户任务调度的时间成本，并能够确保用户任务调度的安全性。
由于云计算任务调度问题与传统分布式环境下的任务调度具有一定的相似性，因此一些经典的应用于传统分布式环境下的调度算法也可以为云计算任务调度问题所借用。如最小执行时间算法（Minimum Execution Time, MET）、最小完成时间算法（Minimum Completion Time, MCT）、Min-min算法、Max-min算法[6]等，这些算法都可以用于云计算的任务调度问题。
现有的任务调度策略没有考虑车载云计算中的动态数据传输速率，大多数是基于任务执行时间进行任务调度，并且对于车载云节点与用户之间的动态连接问题，也没有考虑。因此现有的云计算任务调度算法并不适用车载云计算系统，为此，本课题将针对车载云环境下的动态传输特性，研究能够满足截止时间并具有最小服务成本的任务调度方案。
